# HJMedia 四周学习计划大纲

## 总览

HJMedia 是花椒直播开源的跨平台 C++ 多媒体框架（当前版本 **v1.0.7**），核心产品线包括：

- **推流器（Pusher）**：采集 → 处理 → 编码 → RTMP 推流
- **播放器（Player）**：解复用 → 解码 → 渲染 / 播放
- **音乐播放器（MusicPlayer）**：纯音频源 → 解复用 → 解码 → 重采样 → 音频渲染
- **混音器（AudioMixer）**：多路 PCM 输入 → 混音 → PCM 输出
- **AI 推理 / 萌颜（Inference / FaceU）**：人脸检测 → 萌颜贴纸 → 人脸保护

支持平台：**HarmonyOS（主力）**、Android、iOS、Windows、macOS、Linux。

本计划假设你已具备 C++ 基础，按"先整体、再推流、再播放、最后深入扩展"的路径组织。

## 第一周：项目认知与构建体系

**目标**：跑通编译，理解顶层架构，建立代码浏览的"地图"。

### Day 1-2：项目概览与文档阅读
- 阅读 `README.md`，了解项目定位、特性矩阵（含萌颜、人脸保护、音乐播放器、混音器等新特性）
- 阅读 `CLAUDE.md`，记住各目录职责（新增模块：`src/db`、`src/detect`、`src/gui`、`src/convert`、`src/localserver`）
- 阅读 `CHANGELOG.md`，了解版本演进（v1.0.0 → v1.0.7），把握功能迭代脉络
- 阅读 `docs/architecture/HJGraphMusicPlayer.md`，了解音乐播放器架构
- 阅读 `src/plugins/doc/README.md`，了解插件文档体系和推荐阅读路径
- 阅读 `docs/README_HJOHDemuxer.md` 和 `docs/README_HJOHMuxer.md`，了解 HarmonyOS 侧复用器/解复用器设计

### Day 3-4：构建系统与第三方依赖
- 阅读 `CMakeLists.txt`（根目录），理解三层组织：third_party → src → examples
- 阅读 `CMakePresets.json`，了解各平台 preset 配置（4 个 configure preset + 3 个 build preset）
- 理解 CMake 关键选项：`HJ_ENABLE_TNN`、`HJ_ENABLE_RENDER_PRIO`、`DISABLE_HJPLAYER`、`DISABLE_HJPUSER`、`DISABLE_HJRENDER`、`DISABLE_HJINFERENCE`
- 阅读 `DEPS` 文件，了解第三方依赖管理（depsync 工具 + 补丁系统）
- 阅读 `apply_patches.sh`，理解补丁应用机制（15 个第三方库补丁）
- 运行 `./sync_deps.sh` 同步第三方依赖
- 尝试完成一个平台的编译（推荐 Windows 或 HarmonyOS，也可用 `build_android.bat` 体验 Android 编译）

### Day 5-7：核心架构初探
- **`src/core`**：阅读 `HJContext`、`HJGraphBase`、`HJGraphDec`、`HJMediaNode`，理解"图-节点"模型；阅读 `HJPlayerBase` / `HJMediaPlayer` 理解播放器状态机
- **`src/graphs`**：浏览所有 Graph 类型（HJGraphPusher、HJGraphLivePlayer、HJGraphVodPlayer、HJGraphMusicPlayer、HJGraphAudioMixer），理解不同场景的图编排差异
- **`src/media`**：浏览媒体模型/类型定义，了解 capture、codec、demuxer、muxer、render、net、datasource、io、sei 等子模块
- **`src/plugins`**：浏览平台插件布局（hsys=Harmony、asys=Android、isys=iOS、wsys=Windows），理解插件系统架构
- **`src/comp`**：了解组件层 5 个子模块——graphic（OpenGL 图形）、prio（前处理渲染管线）、rte（实时渲染引擎）、utils（人脸点位/Faceu）、include
- **新增模块概览**：
  - `src/db`：基于 sqlite_orm 的媒体数据库管理
  - `src/detect`：多后端 AI 人脸检测（ncnn / tnn / mindspore / CoreML / Vision）
  - `src/gui`：基于 ImGui 的桌面 UI 框架（仅 Windows/macOS）
  - `src/localserver`：本地 HTTP 缓存服务
  - `src/convert`：格式转换工具
  - `src/entry` 扩展：render（渲染入口）、inference（AI 推理入口）、inferenceRender（Harmony 推理+渲染整合）、hsys（Harmony 通用入口）
- 画出顶层模块依赖图

### 第一周产出
- 成功编译至少一个平台
- 能口述项目三层架构及主要模块职责
- 知道"图(Graph) + 节点(Node) + 插件(Plugin)"流水线模型是什么
- 能列出至少 5 个 Graph 类型及其使用场景

## 第二周：推流器（Pusher）流水线

**目标**：深入理解采集 → 处理 → 编码 → RTMP 推流的完整链路。

### Day 1-2：入口与图编排
- **`src/entry/pusher`**：阅读 Pusher 入口代码（hsys + bridge + verify），了解对外 API 如何映射到内部调用
- **`src/entry/hsys`**：阅读 HJEntryBaseRender、HJNativeExportCommon，理解 Harmony NAPI 导出机制
- **`src/graphs/HJGraphPusher`**：理解推流图如何组装各节点

### Day 3-4：采集与预处理组件
- **`src/media/capture`**：音视频采集
- **`src/comp/prio`**：前处理管线（Prio）
  - HJPrioComFaceu：萌颜贴纸处理
  - HJPrioComFBOBlur：模糊处理（人脸保护场景）
  - HJPrioComFBOGray：灰度处理
  - HJPrioComGiftSeq：礼物序列
  - HJPrioComSourceBridge / SourceSeries / SplitScreen：源桥接/序列/分屏
  - HJPrioGraph / HJPrioGraphProc：前处理图编排
- **`src/comp/rte`**：实时渲染引擎
  - HJRteComDraw：实时绘制
  - HJRteComDrawSRFilter：超分滤波
  - HJRteComDrawDenoiseFilter：降噪
  - HJRteComSourceFaceu：Faceu 输入桥
  - HJRteGraph / HJRteGraphProc / HJRteGraphSetupInfo：RTE 图编排
- **`src/comp/graphic`**：底层 OpenGL 支撑（Shader、FBO、PBO、EGL Surface）
- **`src/comp/utils`**：人脸点位管理（HJFacePointMgr、HJFacePointMadeup）和 Faceu 信息
- **`src/detect`**：多后端人脸检测（hsys/mindspore、ncnn、tnn、osys/coreml、osys/vision）
- 理解"漂流瓶方式驱动"的状态切换机制

### Day 5-6：编码与输出
- `HJComAudioEncoder`（AAC 编码，支持 fdk-aac）
- `HJComVideoEncoder`（H.264/H.265 硬编码，HarmonyOS 支持 ROI）
- `HJComInterleave`：音视频交织
- `HJComRTMP`：RTMP 推流（含重试机制、网络拥塞丢包策略）

### Day 7：推流器录制与回顾
- `HJComMuxer`：MP4 录制（推流中随时短视频录制）
- 回顾 Pusher 完整数据流：采集 → 人脸检测/萌颜前处理(可选) → 编码 → 交织 → RTMP 输出 / MP4 录制
- 了解 MessageBus 模块（v1.0.7）：Graph 和 Plugin 之间的类型安全同步消息调用

### 第二周产出
- 能在源码中追踪一条完整推流路径（含萌颜前处理分支）
- 理解各组件（Com）的职责和连接方式
- 能说明 prio 和 rte 两条前处理管线的区别和适用场景
- 理解人脸检测的多后端架构

## 第三周：播放器（Player）流水线

**目标**：深入理解多种播放场景的解复用 → 解码 → 渲染链路。

### Day 1-2：入口与图编排
- **`src/entry/player`**：阅读 Player 入口代码（asys + hsys + isys + bridge + verify），理解多平台入口差异
- **`src/core/HJMediaPlayer`**：理解播放器核心状态机
- **`src/graphs`**：找到所有 Player 相关 Graph：
  - HJGraphLivePlayer（直播播放）
  - HJGraphVodPlayer（点播播放，支持暂停/恢复/Seek/变速/按指定次数重复播放）
  - HJGraphMusicPlayer（纯音频播放，支持 seek/repeat/EOF）
  - HJGraphAudioMixer（多路混音）

### Day 3-4：解复用与解码
- `HJComDemuxer`：解复用（HTTP-FLV、RTMP、M3U8，含带切换的 M3U8 素材）
- `HJComVideoDecoder`：视频解码（H.264/H.265 硬解+软解，支持格式无缝切换）
- `HJNodeADecoder`：音频解码（FDK-AAC 软解 + 鸿蒙音频硬解）
- `HJComMediaFrameCvt`：帧格式转换
- **`src/convert`**：格式转换工具层

### Day 5-6：渲染与播放控制
- `HJComVideoRender`：视频渲染（透明通道视频礼物支持）
- 播放控制：变速播放、动态追帧、预加载（HTTP 预连接 TLS+TCP+DNS 连接池）、网络失败自动重连
- 多实例管理、音量控制
- **`src/localserver`**：本地 HTTP 缓存服务（支持边下边播）
  - HJCacheManager / HJCacheFile：缓存管理
  - HJBlockManager / HJBlockFile：块文件管理
  - HJDownloader：HTTP 下载器
  - HJHTTPServer / HJLocalServer：本地 HTTP 服务器

### Day 7：播放器高级特性
- 萌颜后处理（播放端）
- SEI 解析与回调（`docs/sei_handling_design.md`）
- 播放质量监控与数据上报
- 音频数据回调、视频数据回调
- 音乐播放器深度：音频链（demuxer → decoder → resampler → render）、时间轴处理（HJTimelineHandler）
- 混音器深度：多路输入音量/静音控制
- 统计与日志

### 第三周产出
- 能在源码中追踪至少 3 种播放路径（直播、点播、纯音频）
- 理解播放器状态机和 MessageBus 消息机制
- 能对比 Pusher 和 Player 在图编排上的异同
- 理解本地 HTTP 缓存服务的架构和边下边播原理
