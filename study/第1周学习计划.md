## 第1天：了解项目背景与整体定位

### 学习目标
- 明确 HJMedia 项目解决的问题和主要使用场景。
- 初步理解 pusher 与 player 两条核心产品链路，以及 1.0.6+ 新增的音乐播放器和混音器。

### 学习内容
1. 阅读仓库根目录的 `README.md`，了解项目定位和完整特性矩阵。
2. 阅读 `CHANGELOG.md`，了解 v1.0.0 → v1.0.7 的版本演进，把握功能迭代脉络。
3. 了解 HJMedia 的核心定位：跨平台 C++ 多媒体框架（HarmonyOS 主力，支持 Android/iOS/Windows/macOS/Linux）。
4. 重点理解五条主要链路：
   - **Pusher**：采集 → 人脸检测/萌颜前处理(可选) → 编码 → 封装/交织 → RTMP 输出
   - **Player**：RTMP/HTTP-FLV/M3U8 输入 → 解复用 → 解码 → 萌颜后处理(可选) → 渲染
   - **MusicPlayer**：音频源 → 解复用 → 解码 → 重采样 → 音频渲染（支持 repeat/seek/EOF）
   - **AudioMixer**：多路 PCM 输入 → 独立音量/静音控制 → 混音 → PCM 输出
   - **Inference/FaceU**：人脸检测 → 萌颜贴纸动态跟随 / 人脸保护画面模糊
5. 记录项目中反复出现但暂时不熟悉的关键词。

## 第2天：熟悉顶层目录结构

### 学习目标
- 建立对仓库目录结构的整体印象。
- 知道主要代码模块大致负责什么。

### 学习内容
1. 浏览仓库顶层目录。
2. 重点了解以下目录的作用：

| 目录 | 主要职责 |
|------|----------|
| `src/core` | 核心运行时、图和节点基础抽象、播放器状态机、时间轴处理 |
| `src/media` | 媒体类型、数据结构（capture/codec/demuxer/muxer/render/net/datasource/io/sei） |
| `src/comp/graphic` | OpenGL 图形组件（Shader/FBO/PBO/EGL Surface） |
| `src/comp/prio` | 前处理渲染管线（萌颜/模糊/灰度/礼物序列/分屏） |
| `src/comp/rte` | 实时渲染引擎（超分/降噪/Faceu 桥接） |
| `src/comp/utils` | 组件工具（人脸点位管理/Faceu 信息/图片序列） |
| `src/plugins` | 插件式后端实现（hsys=Harmony/asys=Android/isys=iOS/wsys=Windows） |
| `src/graphs` | 端到端媒体流水线图编排（Pusher/LivePlayer/VodPlayer/MusicPlayer/AudioMixer） |
| `src/entry/pusher` | 推流入口模块 |
| `src/entry/player` | 播放入口模块 |
| `src/entry/render` | 渲染入口模块 |
| `src/entry/inference` | AI 推理入口模块 |
| `src/entry/inferenceRender` | Harmony 推理+渲染整合入口 |
| `src/entry/hsys` | Harmony 通用入口（NAPI 导出） |
| `src/detect` | 多后端 AI 人脸检测（ncnn/tnn/mindspore/CoreML/Vision） |
| `src/db` | 媒体数据库管理（基于 sqlite_orm） |
| `src/localserver` | 本地 HTTP 缓存服务（缓存/下载/块文件管理） |
| `src/gui` | 桌面 GUI 框架（基于 ImGui，仅 Windows/macOS） |
| `src/convert` | 格式转换工具 |
| `src/utils` | 基础工具（线程/平台工具/HJThread） |
| `src/stat` | 状态统计 |
| `src/datafuse` | 数据融合 |
| `src/links` | 链接/传输层 |
| `src/platform` | 平台抽象（共享内存） |
| `third_party` | 第三方源码依赖（20+ 个库） |
| `externals` | 按平台分组的预编译第三方库 |
| `patches` | 第三方库补丁文件（15 个） |
| `docs` | 架构/设计/产品/交付文档 |
| `examples/harmony` | HarmonyOS 示例和 API 集成 |
| `examples/windows` | Windows 演示/工具项目 |
| `examples/android` | Android SDK 封装 |
| `examples/ios` | iOS 演示 |
| `demo/faceu` | 萌颜 4 平台演示 |
| `study` / `studyNote` | 学习计划和笔记 |
| `cmake` | CMake 模块（40+ 个 FindXXX / Config / 辅助脚本） |

3. 初步理解项目不是单文件式流水线，而是由多个模块共同完成媒体处理。

## 第4天：学习音视频基础概念

### 学习目标
- 建立音视频开发的基础术语体系。
- 能够把代码中的媒体对象和真实音视频概念联系起来。

### 学习内容
1. 学习视频基础概念：
   - 帧、分辨率、帧率、码率、关键帧、GOP。
   - YUV 色彩空间、H.264 vs H.265 区别。
2. 学习音频基础概念：
   - 采样率、声道数、采样格式、码率、音频帧。
   - AAC 编码、PCM 原始音频、音频重采样。
3. 学习时间相关概念：
   - PTS、DTS、时间戳、音画同步、延迟。
4. 学习处理流程概念：
   - 编码、解码、封装、解封装、推流、拉流、渲染。
   - 交织（Interleave）：音视频交错打包。
   - 混音（Audio Mixer）：多路音频混合。
5. 学习常见格式和协议：
   - 视频编码：H.264 / H.265。
   - 音频编码：AAC。
   - 封装/容器：FLV、MP4、M3U8（含带切换的变体）。
   - 传输协议：RTMP、HTTP-FLV、HLS（M3U8）。
6. 了解 HJMedia 特有概念：
   - **SEI**：视频流中的补充增强信息，可携带自定义数据。
   - **萌颜（FaceU）**：基于人脸关键点的动态贴纸渲染。
   - **人脸保护**：检测到无人脸时自动模糊画面。
   - **超分（Super Resolution / SR）**：通过 AI 模型提升画面分辨率。
   - **ROI 编码**：对感兴趣区域（如人脸）使用更高质量编码参数。

### 实践任务
- 制作一份音视频关键词表，每个关键词用 1 到 3 句话解释。
- 分别写出"摄像头画面推到 RTMP 服务器"和"从地址播放视频"的高层流程。
